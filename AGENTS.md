# AGENTS.md

## 项目工作规则

- 新会话启动读取规则与交接更新规则，统一以本文件为准。

### 1）启动与基线规则
- 每次新会话首次回复前，先读取仓库根目录的 `handoff.md`。
- 后续实现类工作都以 `handoff.md` 作为当前状态基线。
- 读取完成后，首次回复首句固定为：`项目及进展情况已了解，我们可以继续工作了。`

### 2）强制更新规则
- 只要本轮修改了仓库文件，必须在同一轮更新 `handoff.md`。
- 如果本轮仅聊天（无文件修改），不要修改 `handoff.md`。

### 3）哪些行为算“仓库修改”
- 代码修改
- 配置修改
- 脚本修改
- 文档修改
- 新增/删除文件
- 与文件内容绑定的行为变化

### 4）修改后必须刷新 `handoff.md` 的区块
- `最后更新时间`
- `本轮改动（按时间）`
- `当前项目状态`
- `已确认决策`
- `未解决问题 / 风险`
- `下一步`
- `常用命令`

### 5）`handoff.md` 写作要求
- 简洁、事实化、可执行。
- 优先写具体路径和可直接运行的命令。
- 需要重启服务时必须写明。
- 有失败尝试时，记录失败点和原因。

### 6）优先级规则
- 若 `handoff.md` 与旧对话上下文冲突，以“当前代码库状态 + 用户最新指令”为准，并同步更新 `handoff.md`。
